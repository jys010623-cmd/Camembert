# 까망베르 (Camembert) — Desktop Companion
### 캐릭터 IP 기획부터 Electron 앱 구현까지, 1인 개발 프로젝트

## 프로젝트 개요
> "곁에 있어주는 존재"를 만들고 싶었습니다.

**까망베르**는 바탕화면 한켠에 앉아 사용자와 함께 있어주는 데스크톱 펫입니다. 리듬게임 서비스 '리듬베르'의 공식 마스코트를 IP로 설정하고, 그 캐릭터를 실제로 살아 움직이게 만드는 것이 목표였습니다.
캐릭터 설정(바이블) → 에셋 명세 → 애니메이션 엔진 → 행동 AI → 시스템 연동까지 **기획·디자인 방향·개발 전 과정을 혼자** 진행했습니다.

- **역할**: 기획 · 캐릭터 설계 · 프론트/앱 개발 (1인)
- **규모**: TypeScript 약 4,300줄 / 32개 모듈 / Electron 데스크톱 앱
- **기술 스택**: `Electron` · `TypeScript` · `Vite` · Canvas 스프라이트 · Web Audio API · IPC

---

## 1. 캐릭터 기획 — 정체성을 문서로 고정하다
단순히 "귀여운 캐릭터"를 넘어, **어떤 매체(앱·웹·굿즈·SNS)에서도 동일한 캐릭터**가 유지되도록 캐릭터 바이블을 먼저 썼습니다.

- **컨셉**: 2등신 검은 고양이 플러시 정령. 소리에 이끌려 데스크톱으로 찾아와, 사용자의 작업 리듬에 맞춰 조용히 함께 있어주는 존재
- **성격 7축** (조용함·귀여움·순함·호기심·음악애호·응원가·자족적)을 정의하고, 이를 **에너지 30% / 표현강도 25%** 같은 '성격 온도계'로 수치화해 연출 기준을 만듦
- **말투 규칙**: 짧고 부드럽게, 과장된 리액션 금지
- **🔒 고정 요소 vs 🎨 가변 요소**를 명시 — 얼굴 비율·눈 색(`#B9A7FF`)·팔레트는 절대 불변, 표정·포즈·소품만 변형 허용

> 이 '무엇을 바꿀 수 있고 무엇은 절대 안 되는가'라는 기준이, 이후 애니메이션 에셋과 코드 구조 전체의 설계 근거가 되었습니다.

## 2. 기능 — 살아있는 것처럼
- **투명·프레임리스·항상 위** 창에 캐릭터가 떠 있고, 드래그로 옮기고 클릭하면 반응
- **상태 머신 기반 행동 엔진** — 대기 / 감정 / 음악 / 춤 / 수면 / 걷기 / 방향전환 상태를 오가며, 아무도 건드리지 않아도 스스로 랜덤하게 행동 (하품 → 꾸벅꾸벅 → 잠 같은 자연스러운 흐름)
- **다마고치식 스탯** — 배고픔·에너지·기분이 시간에 따라 변하고 행동 선택에 반영 (피곤하면 더 자주 존다). 먹이주기·놀아주기로 케어
- **말풍선 대사** — 캐릭터 바이블의 말투 규칙을 그대로 코드화한 한국어 대사 뱅크 (인사·클릭·상태 변화별)
- **시스템 연동** — 마이크로 음악을 감지하면 비트에 맞춰 춤, 커서 따라다니기, 화면 가장자리에 붙어 쉬기, 정시 알림·배터리 인식

---

## 3. 기술적으로 공들인 부분
- **알파 픽셀 히트테스트** — 2048×2048 원본에서 캐릭터의 불투명 영역만 자동 크롭하고, 실제 보이는 픽셀 단위로 클릭·드래그를 판정. 투명한 여백을 눌러도 펫이 잡히지 않음
- **공용 크롭 박스 설계** — 모든 애니메이션 클립의 불투명 영역을 합집합으로 계산해 하나의 기준 박스로 통일. 동작이 바뀔 때 캐릭터가 커졌다 작아졌다 튀는 문제를 근본적으로 해결
- **레이어 충돌 방지** — 따라다니기·가장자리 붙기·마이크 댄스 같은 모션이 자율 행동과 싸우지 않도록, 활성화되면 자동으로 자율 루프를 일시정지
- **에셋 없이도 동작하는 파이프라인** — 프레임 이미지가 없어도 placeholder 타이밍으로 전체 행동 루프가 돌아, 기획·아트·개발을 병렬로 진행
- **3계층 아키텍처** — main(창·트레이·에셋 스캔) / preload(보안 브리지) / renderer(행동 엔진) 분리, 폴더에 PNG만 넣으면 재빌드 없이 자동 반영

## 마무리 — 배운 것
캐릭터라는 '감성'을 규칙과 수치로 번역해 코드로 구현하는 과정에서, **기획 문서가 곧 아키텍처의 설계도가 된다**는 걸 체감했습니다. "무엇을 고정할지 먼저 정하는 것"이 확장 가능한 시스템을 만드는 핵심이었습니다.
